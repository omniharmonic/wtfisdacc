# Execution Environments

**Category**: Infrastructure > Execution
**D/ACC Score**: 3 | **Investment Tier**: [[Tier 3 - Selective Conviction]]
**Development Stage**: Growth | **Risk Profile**: Medium
**Defense Category**: [[Cyber Defense]]

## Overview

Virtual machines for smart contract computation with varying performance/decentralization tradeoffs. The EVM (Ethereum Virtual Machine) maintains 70%+ developer share, creating strong network effects.

## Why D/ACC Aligned

Open-source execution prevents proprietary lock-in. Multiple VMs enable ecosystem redundancy—if one environment is compromised or captured, alternatives exist.

## Key Implementations

| VM | Ecosystem | Notes |
|----|-----------|-------|
| EVM | Ethereum, L2s | 70%+ developer share |
| Solana VM | Solana | High throughput |
| Move VM | Aptos, Sui | Resource-oriented |

## D/ACC Considerations

Lower score (3) because:
- EVM dominance creates concentration risk
- Developer network effects may prevent alternatives from emerging
- Proprietary optimizations in some implementations
