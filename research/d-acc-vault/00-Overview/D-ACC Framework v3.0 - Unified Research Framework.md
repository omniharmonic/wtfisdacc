# D/ACC Framework v3.0 — Unified Research Framework

*A Theoretically Sound, Empirically Grounded Framework for Defensive Accelerationism*

**Version**: 3.0
**Date**: January 23, 2026
**Status**: Final

---

## Executive Summary

This framework synthesizes Vitalik Buterin's Defensive Accelerationism (d/acc) philosophy into a coherent system for understanding, evaluating, and investing in technologies that shift the balance of power toward defense, decentralization, and democratic participation.

The framework operates on three integrated layers:

1. **Philosophical Foundation** — Vitalik's Quadrant (Atoms/Bits × Survive/Thrive) + Four D's evaluation criteria
2. **Technology Domains** — Seven empirically-validated categories emerging from d/acc community practice
3. **Implementation Primitives** — Six functional building blocks that enable d/acc technologies

Each layer informs the others, creating a unified model that is both theoretically rigorous and practically applicable.

## Part I: Philosophical Foundation

### The Core Thesis

> "The core idea of d/acc is simple: decentralized, democratic, and differentiated defensive acceleration. Build technology that can tip the balance of attack and defense toward defense, and do so without relying on giving more power to a central authority." — Vitalik Buterin

Technology is not neutral. The current trajectory accelerates:
- Surveillance over privacy
- Centralization over distribution
- Offense over defense
- Authoritarian control over democratic participation

D/ACC inverts this trajectory by deliberately accelerating technologies with defensive, decentralized, democratic, and differential properties.

### The Four D's Evaluation Criteria

Every technology is evaluated against four criteria, each representing a core d/acc principle:

#### 1. DEFENSIVE
*Does this shift the offense/defense balance toward protection?*

| Score | Level | Characteristics |
|-------|-------|-----------------|
| 21-25 | **Strongly Defensive** | Fundamentally defensive; offense impossible (e.g., Far-UVC kills all pathogens) |
| 16-20 | **Defensive** | Clear defensive advantage; hard to weaponize (e.g., ZK proofs) |
| 11-15 | **Mildly Defensive** | Some defensive properties with offensive potential |
| 6-10 | **Neutral** | Equally useful for offense and defense |
| 0-5 | **Offensive** | Creates attack surfaces; enables cheaper harm |

**Quintessential Example**: Far-UVC light kills ALL pathogens universally without targeting specific threats. Pure defense—no offensive use possible.

#### 2. DECENTRALIZED
*Can this operate without requiring trust in centralized actors?*

| Score | Level | Characteristics |
|-------|-------|-----------------|
| 21-25 | **Maximally Decentralized** | Credibly neutral; resistant to capture; permissionless |
| 16-20 | **Decentralized** | No single point of failure; many independent operators |
| 11-15 | **Distributed** | Multiple operators but some concentration |
| 6-10 | **Federated** | Identifiable, coordinated set of parties |
| 0-5 | **Centralized** | Single point of control |

**Key Insight**: Requires trust that "good guys" will stay good = NOT decentralized.

#### 3. DEMOCRATIC
*Does this enable broader participation and resist capture by small groups?*

| Score | Level | Characteristics |
|-------|-------|-----------------|
| 21-25 | **Deeply Democratic** | Equal voice; Sybil-resistant; cross-partisan |
| 16-20 | **Participatory** | Meaningful participation available to most |
| 11-15 | **Representative** | Delegation with capture potential |
| 6-10 | **Oligarchic** | Concentrated influence despite some participation |
| 0-5 | **Plutocratic** | Wealth equals power |

**Quintessential Example**: Pol.is surfaces cross-partisan consensus without pre-assumed authorities deciding truth.

#### 4. DIFFERENTIAL
*Does defense grow faster than offense over time?*

| Score | Level | Characteristics |
|-------|-------|-----------------|
| 21-25 | **Asymmetric** | Fundamentally favors defense; offense gains nothing from improvements |
| 16-20 | **Strong Differential** | Defense scales superlinearly vs. offense |
| 11-15 | **Moderate Differential** | Defense compounds faster |
| 6-10 | **Slight Edge** | Defense benefits marginally more |
| 0-5 | **Arms Race** | Improvements benefit both equally |

**Key Insight**: Network effects that favor defense create differential advantage.

## Part IV: The Evaluation Rubric

### Overview

Projects are evaluated on two dimensions:
1. **D/ACC Alignment Score** (0-100): Four D's × 25 points each
2. **Market Opportunity Score** (0-100): Size × Growth × Timing × Competition

### D/ACC Alignment Scoring

| Criterion | 0-5 | 6-10 | 11-15 | 16-20 | 21-25 |
|-----------|-----|------|-------|-------|-------|
| **Defensive** | Offensive | Neutral | Mildly Defensive | Defensive | Strongly Defensive |
| **Decentralized** | Centralized | Federated | Distributed | Decentralized | Maximally Decentralized |
| **Democratic** | Plutocratic | Oligarchic | Representative | Participatory | Deeply Democratic |
| **Differential** | Arms Race | Slight Edge | Moderate | Strong | Asymmetric |

### Market Opportunity Scoring

| Criterion | 0-5 | 6-10 | 11-15 | 16-20 | 21-25 |
|-----------|-----|------|-------|-------|-------|
| **Market Size** | <$100M | $100M-$1B | $1B-$10B | $10B-$100B | >$100B |
| **Growth** | Negative | 0-10% | 10-25% | 25-50% | >50% CAGR |
| **Timing** | 10+ years | 5-10 years | 3-5 years | 1-3 years | Now |
| **Competition** | Commoditized | Competitive | Differentiated | Category Leader | Essential Infrastructure |

## Part VI: Framework Summary

### The Three-Layer Model

```
┌─────────────────────────────────────────────────────────────────────────┐
│  LAYER 1: PHILOSOPHY                                                     │
│  Quadrant (Atoms/Bits × Survive/Thrive) + Four D's (Defensive,          │
│  Decentralized, Democratic, Differential)                                │
├─────────────────────────────────────────────────────────────────────────┤
│  LAYER 2: DOMAINS                                                        │
│  7 Technology Areas: Biosecurity, Physical Resilience, Cyber Defense,   │
│  Info Resilience, Social Tech, NeuroTech, Decentralized AI              │
├─────────────────────────────────────────────────────────────────────────┤
│  LAYER 3: PRIMITIVES                                                     │
│  6 Building Blocks: COMPUTE, COORDINATE, EXCHANGE, VERIFY, STORE,       │
│  CONNECT                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

### Why This Framework Works

1. **Philosophically Grounded**: Directly derived from Vitalik's writings
2. **Empirically Validated**: Categories match actual d/acc community practice
3. **Mutually Exclusive**: Each project has one primary domain
4. **Collectively Exhaustive**: All d/acc projects fit somewhere
5. **Practically Applicable**: Rubric enables consistent evaluation
6. **Investment-Ready**: Tier system guides capital allocation

### Key Insights

1. **Privacy is a technique, not a domain**: ZK, FHE, MPC are tools used across domains
2. **Domains answer WHAT; Primitives answer HOW**: Every project has a domain (biosecurity, info defense, etc.) and uses primitives (VERIFY, COORDINATE, etc.)
3. **The Four D's are not binary**: Each is a spectrum (0-25), enabling nuanced evaluation
4. **Market opportunity matters**: High D/ACC alignment with no market = Tier 2 (monitor), not Tier 1
